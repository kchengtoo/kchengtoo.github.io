DrawBot is a CNC robot designed for people who aspire to pursue in the field of engineering by transforming code into illustrations. As it is highly customizable, it provides the tools for engineers to understand how machines and program work.

This case study will only focus on the design of DrawBot. To see the technology behind this project, click on tech to check it out.

2019 - Robotics
Pencil on paper, Illustrator, Photoshop, Rhinoceros 3D

Problem

Whether you want to learn how to code or engineer, it can be a huge leap for those who do not have a background in either fields. Speaking from experience, the sheer number of concepts that you have to learn from programming can be overwhelming, which can discourage many people from continuing their dream to become an engineer. However, we can overcome this challenge by providing the means to make a smooth transition that will ease their concerns.

Everyone can engineer.

Process

In order to create a learning tool that people will use, I have to understand what users want to get out from a learning tool. Once the research gives me some insights, I will brainstorm ideas for the design, and eventually, create a prototype for testing to see if I missed anything that the users are looking for.

Designing and developing a learning tool within six weeks.

Research

My research began by looking at examples of learning tools to better understand what it should be, and Scratch was my prime example. Scratch is a block-based visual language that is designed to teach children the fundamentals and logic of programming. The program is designed to be fun and engaging, and I want to apply some of that aspect to my design.

Teaching programming fundamentals and logic through Scratch.

I launched my user research by sending out surveys to engineering students with different levels of experience, from beginner to professional. Since, I'm designing a learning tool for aspiring engineers, I was looking for students who struggled to learn engineering when they started off. I was able to identify 3 participants to move forward with the user interviews.

Insight

After the interviews, I began to dissect the information and establish the obstacles that aspiring engineers have as they started on this path. Here are some the concerns that they have:

  • Students have trouble thinking like a program, e.g. logic and concepts.
  • Being able to determine the issue with the program and debug.
  • Better understand how to assemble hardwares and create a device.

Design

When it comes to design a learning tool, the most important factor is the motivation to use. Scratch had the right idea of teaching programming by creating an engaging yet easy-to-use interface. However, in addition to programming, I wanted this tool to be able to teach other engineering skills, such as circitry and hardwares. Thus, here were some of the components that I need in order to create this learning tool:

  • Develop a program that users can modify and experiement with code.
  • Build a configurable device that runs and outputs code in real-life.
  • Design a protective case for the device that also acts as the "face" of the tool.

Program Design

I used Arduino to create this tool, because Arduino has both software and hardware components that allowed me to build a digital device. While there were no visual components of the program, there were a couple of design considerations:

Action Functions

Program basic functions, such as move forward, pen down and turning off device.

Instructions

Note instructions in the program to offer user guidance when modifying the program.

Illustration Example

Provide an illustration example, so users can get an idea what this tool can do.

Device Design

I designed the device to output code in real-life by drawing it out, because I believe that it helps users better visualize and understand how a program execute code. In order to create a drawing robot, I created a chassis that holds the Arduino hardwares. Furthermore, since this tool allows users to play with the hardwares, the components are not soldered in place.

Chassis designed by MakersBox.

Shell Case Design

The primary purpose of the shell case is to protect the device, but we took the opportunity designing the shell to make the tool look more engaging. I drew sketches of the shell with the intention to look more futuristic, so a couple of the designs have robotic expressions.

My sketches of the shell case design.

I decided to go with the design with the trinity-shaped expression, because it's a very robotic design that suited the purpose of a tool. The shell was made out of white illustrator board with graphics printed on it.

Shell case with the trinity-shaped experession.

Usability Testing

I conducted user testings with five participants (three male and two female; average age 20.2 years; all major in engineering) to see if they find the learning tool intuitive to use and if the design of the shell case was appealing for its purpose. The participants were able help us highlight issues that I needed to address in the next iteration:

  • Implement wireless connection when uploading the code to the program. The new iteration of the program and device can be found in tech.
  • Design the shell case to show some kind of personality, so users can feel connected to it.

My sketches for second iteration of the shell case.

I took inspirations of robot design from pop cultures that became popular due to their appeal and personality. I loved Wall.E and Baymax's designs, so I created a design that encompasses characteristics from these lovable robots.

Final shell case design.

Prototype

Once all the components are completed, I assembled them all together and created a prototype to help you get an idea what DrawBot can do for the good of education.

DrawBot executing code by drawing it out.

Conclusion

As one of the earliest projects that integrated design with technology, it built a foundation for future projects with similar elements. DrawBot is a prototype of a learning tool for aspiring engineers and I want to find the best way to make that happen. If I had more time, I would like to conduct more rounds of user testing, so I can better understand what users need when using an educational tool to learn something new. That being said, I hope this iteration of DrawBot will help the future generations of aspiring engineers.

DrawBot exhibited at the 2019 USC Roski Annual Student Exhibition.
